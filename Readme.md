# Readme.md 

엑셀 쿼리를 활용한 조합 필터링   

여기에 쓰는 내용은 프로그램을 작성하면서 **코드에 대해 느낀점**이다.    

우선 아래 두 개는 사람들과 이야기를 하면서 느낀점이다.  

* 고객들은 자기들이 요구하는 요구사항이 무엇인지 잘 모르는 경우가 많다. 그래서 결과물을 보고 말이 바뀌는 경우가 많다.  

*  경우의수가 어느정도 될지 가늠은 하지만 그것을 동작하는데 요구되는 시간이 어느 정도일지 모른다.  
그리고 자신이 요구하는 쿼리가 몇 가지 경우의 수가 나오고, 얼마만큼의 컴퓨터 자원이 필요하는지는 전혀 알지 못 한다.  

두 번째의 경우는 확실히 개발자의 책임이다.  

시간이 지나고 보니, 나는 첫 번째의 경우도 개발자의 책임이라고 생각한다.  
의뢰인들은 대개 컴퓨터를 잘 알지 못한다. 그리고 상대적으로 프로그램이 어떤 방식으로 돌아가는지에 대한  
자연어로 표현하는 역량이 프로그래머보단 낮다.  

따라서, 미리 고객의 요구사항이 종종 변한다는 것에 대해 마음의 준비를 하고, 코드에 녹여내야한다.   

이번에는 어떤 행위 하나 하나마다 추상화하기 위해 클래스를 많이 만들게 되었다.   

그런것을 방지하기위해 내 프로그램을 짤 때 열어둘 수 있는 가능성들은 열어두고 일반화할 수 있는 
행위들은 일반화 해야한다.  

그리고 다음에 소스를 읽을 나를 위해서 최대한 깔끔하게 써야하고, 원시 데이터 타입을 최대한 캡슐화 해줘야 한다.   
다음의 나를 위해서 분기문을 최대한 쓰지 말아야한다. 분기문은 두뇌를 피곤하게 한다... 고 생각한다. (알고리즘 제외)   

이 절차를 최대한 지키려고 노력했다.
